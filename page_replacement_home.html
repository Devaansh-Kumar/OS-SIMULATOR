Page Replacement Algorithms



FIFO - First In First Out 

    FIFO (First-In-First-Out) is a page replacement algorithm in which the page that has been in the memory the longest is the one selected for replacement when a page fault occurs. It follows the principle of treating pages in the order they arrive, resembling a queue data structure.

Advantages of FIFO


1 - FIFO is easy to understand and implement.
2 - The algorithm requires only a simple queue to keep track of the page order.

Disadvantages of FIFO

1 - FIFO can suffer from an increasing the number of frames leading to an increase in page faults
2 - FIFO does not consider the frequency or recency of page accesses, which can result in inefficient use of memory when there is significant reference locality in the page stream.
3 - FIFO treats all pages equally, regardless of their importance or frequency of access, which can lead to suboptimal replacement decisions.




Optimal

    Optimal page replacement algorithm, also known as MIN, is an optimal page replacement strategy that selects the page for replacement which will not be used for the longest duration of time in the future.

Advantages of FIFO


  1 - Optimal algorithm guarantees the fewest number of page faults among all page replacement algorithms, given perfect knowledge of future page accesses. 2 - By considering the future page accesses, optimal replacement algorithm gives an upper bound on the performance that can be achieved by other algorithms.

Disadvantages of FIFO

1 - Optimal page replacement requires knowledge of the entire future sequence of page accesses, which is generally not feasible in real-world scenarios
2 - Implementing the optimal algorithm is challenging since it involves predicting future page accesses accurately, which may require significant computational resources.
3 - Optimal algorithm cannot be used in dynamic or real-time environments where the page access pattern changes frequently, as it assumes a fixed future sequence of page accesses.




LRU - Least Recently Used

    LRU (Least Recently Used) page replacement algorithm selects the page for replacement based on the principle that the least recently used page is the one with the highest probability of being unused in the near future.

Advantages of FIFO


1 - LRU algorithm approximates the optimal page replacement behavior by considering the recency of page accesses, making it effective in many practical scenarios.
2 - LRU tends to keep frequently accessed pages in memory, reducing the number of page faults and maximizing the utilization of available frames. 3).Easy to choose page which has faulted and hasn’t been used for a long time.

Disadvantages of FIFO

1 - Implementing LRU algorithm efficiently can be challenging due to the need for tracking and updating the access timestamps of all pages, which can be resource-intensive. 2 - LRU requires maintaining a data structure, such as a linked list or a priority queue, to track the order of page accesses, which can consume additional memory and introduce overhead for maintaining this data structure.
3 - LRU may struggle in scenarios with rapidly changing or non-uniform page access patterns, as it heavily relies on the assumption that recently used pages are more likely to be used again in the future.

    We need these algorithms to efficiently allocate contiguos memry to processes and try to avoid fragmentation to some extent.

  • What is the advantage of next fit over first fit ?

    Next fit tries to address this problem by starting the search for the free portion of parts not from the start of the memory, but from where it ends last time. Next fit is a very fast searching algorithm and is also comparatively faster than First Fit and Best Fit Memory Management Algorithms.
